## ГЛАВА II. Среда, окружение и изоляция данных

### Раздел 1. Изоляция зависимостей

#### 2.1. Локальный суверенитет окружения
Каждый хобби-проект и скрипт обязан иметь изолированное виртуальное окружение. Глобальная установка сторонних библиотек в операционную систему категорически запрещена. Зависимости проекта должны быть жестко зафиксированы в конфигурационном файле (`requirements.txt`, `Poetry` или `Pipfile`).
* **Факт из истории:** В 2016 году программист Азер Кочулу удалил из глобального репозитория NPM свой крошечный пакет `left-pad` (всего 11 строк кода для выравнивания текста). Из-за того, что тысячи проектов по всему миру не имели жесткой изоляции и бездумно подтягивали этот пакет из глобальной сети, в один момент «сломался» весь интернет-фронтенд, включая сборки Facebook, Babel и Spotify. Локальная фиксация зависимостей внутри проекта — это единственная гарантия того, что ваш код запустится завтра.

#### 2.2. Защита от деградации версий
Запрещено использовать синтаксис или библиотеки, которые жестко привязаны к минорным, экспериментальным версиям интерпретатора. Код должен оставаться стабильным при обновлении базовой среды.
* **Факт из истории:** Переход мирового сообщества с Python 2 на Python 3 занял более 10 лет (с 2008 по 2020 год) и стоил индустрии миллиардов долларов. Из-за отсутствия обратной совместимости в базовых функциях (таких как `print`), миллионы скриптов автоматизации по всему миру превратились в бесполезный цифровой мусор. Писать код нужно так, чтобы он не зависел от сиюминутных капризов создателей языка.

### Раздел 2. Цифровая гигиена и защита секретов

#### 2.3. Тотальное сокрытие ключей и доступов
Ни один пароль, API-токен, приватный ключ или адрес закрытой базы данных не имеет права находиться внутри файлов кода. Все конфигурации сред и секреты выносятся в изолированный файл `.env`, который мгновенно заносится в `.gitignore`.
* **Факт из истории:** В 2019 году компания Uber призналась, что хакеры похитили личные данные 57 миллионов пассажиров и водителей. Злоумышленники не взламывали сложные защитные контуры — они просто нашли приватные ключи доступа к серверам AWS Amazon, которые инженеры Uber по неосторожности оставили прямо внутри кода в одном из публичных репозиториев на GitHub. Оставленный в коде пароль — это заряженное ружье, направленное в автора.

#### 2.4. Режим «Чистого репозитория»
В систему контроля версий (Git) отправляется исключительно чистый исходный код. Временные файлы, логи выполнения скриптов, кэш среды разработки (`.idea/`, `__pycache__/`) и локальные файлы баз данных должны безжалостно отсекаться на подлете.
* **Факт из истории:** В начале 2020-х годов из-за невнимательности разработчиков, случайно загрузивших папки `.git` со служебной информацией на боевые сервера, в открытый доступ утекли исходные коды сотен государственных и банковских систем по всему миру. Git — это архив вашей мысли, а не корзина для мусора и кэша.

### Раздел 3. Порядок в Данных

#### 2.5. Эволюция баз данных через код
Любое изменение структуры локальных таблиц (SQLite, PostgreSQL и др.) фиксируется исключительно в виде SQL-файлов миграций. Внесение изменений «руками» через графические интерфейсы (DBBrowser, DBeaver) запрещено.
* **Факт из истории:** В 2017 году ведущий системный администратор крупнейшего хостинга исходного кода GitLab случайно удалил боевую базу данных во время ручного наведения порядка на сервере. Из-за отсутствия автоматизированных, проверенных кодом скриптов восстановления и миграций, сервис «лежал» больше суток, а часть данных пользователей была безвозвратно утеряна. Любое ручное действие с БД — шаг к потере данных.

#### 2.6. Автономия первого запуска
Скрипт, работающий с базой данных, при первом запуске обязан самостоятельно проверить наличие файла БД или таблиц и автоматически развернуть пустую структуру (сделать *seed* первичных данных), если они отсутствуют. 
* **Факт из истории:** Когда создатели культовой игры *Elite* (1984) писали код для генерации целой галактики из тысяч планет, у них не было места на дискете для хранения этой базы данных. Они написали алгоритм, который создавал всю вселенную «на лету» прямо при запуске игры из одной математической формулы. Ваш скрипт должен быть столь же автономен: запустился — сам создал себе рабочее пространство.
